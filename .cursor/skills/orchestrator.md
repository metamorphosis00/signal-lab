# Orchestrator Skill

## Цель
Разбивать большие задачи на атомарные шаги и выполнять их последовательно.
Экономить контекст — каждый шаг минимальный и независимый.

## Как использовать
Напиши в чате Cursor:
"Используй orchestrator skill для задачи: [описание задачи]"

## Алгоритм работы

### Шаг 1 — Анализ задачи
Перед началом работы:
1. Определи конечный результат — что должно работать в итоге?
2. Разбей на атомарные шаги (каждый шаг — один файл или одна функция)
3. Определи зависимости между шагами
4. Оцени риски каждого шага

### Шаг 2 — Выполнение
Для каждого шага:
1. Объяви что делаешь: "Шаг N: [описание]"
2. Сделай минимальное изменение
3. Проверь что работает
4. Только потом переходи к следующему шагу

### Шаг 3 — Проверка
После каждого шага:
- Запусти приложение
- Проверь что предыдущий функционал не сломался
- Зафикси результат

## Правила экономии контекста
- Один шаг = одно изменение = одна проверка
- Не держи в контексте файлы которые не меняешь
- Используй маленькие модели для простых шагов (генерация boilerplate)
- Используй большие модели только для архитектурных решений

## Пример декомпозиции
Задача: "Добавить новый сценарий memory_leak"

Шаги:
1. Добавить метрику в ScenariosService (1 файл)
2. Добавить метод simulateMemoryLeak() (1 метод)
3. Добавить вызов в runScenario() (1 строка)
4. Добавить кнопку в UI (1 элемент массива)
5. Проверить метрики в /metrics
6. Проверить UI на localhost:3001
